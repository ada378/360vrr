How We Work

From Blueprint to Walkthrough

A four-stage pipeline that turns architectural drawings into explorable environments.

01

Survey

We digitize plans, elevations and material references into a shared spatial reference model.

02

Model

Massing and interior volumes are built to true scale, with lighting studies run per orientation.

03

Render

Each space is rendered as a navigable 360° panorama set, tuned for both desktop and headset viewing.

04

Deliver

Clients receive an embeddable viewer, static stills, and a walkthrough recording for presentations.

Installation

Include the viewer script before your closing body tag. No build step or framework is required — the viewer runs on a plain DOM layer.

<script src="meridian-viewer.js"></script>
// or via a package manager
npm install @meridian/viewer

Embedding a Scene

Create a container with a target id, then initialize the viewer with a scene reference. Scenes can be swapped at runtime without reloading the page.

const viewer = Meridian.mount('#panoStage', {
  scene: 'corten-pavilion',
  autoRotate: true,
  speed: 0.4
});

Configuration

The mount call accepts an options object. Unset values fall back to the defaults below.

OptionTypeDefault
autoRotatebooleantrue
speednumber0.4
dragSensitivitynumber1.0

Controls

Pointer drag pans the camera horizontally across the scene. Release to resume the gentle idle drift. Keyboard arrow keys are supported for accessibility.

Scene Object

Each grid tile in the Work section maps to a scene identifier passed into the viewer. Swapping scenes at runtime looks like this:

viewer.load('atrium-glasshouse');
